The glimpse function is very similar to base R’s str function. We can see that there are 53940 rows in the data set and 11 columns. If we want to see just the number of rows and columns without any of the data we could use the dim function, which takes a single object as an input. The %>% symbol is called the pipe operator. This takes the output of one statement and passes it onto the next statement. In this chunk of code you can read it as “I have the diamonds data, then I use the glimpse function to look at it.”

Dplyr

Dplyr is one of the most important packages included in the tidyverse. It provides functions, or verbs, that allow you to easily manipulate data. In this section we will go over the select, filter, arrange, mutate and summarize verbs. To download a cheat sheet for common dplyr functions click here.

Stringr

Stringr is a package in the tidyverse and is built on top of the stringi package. It plays a big role in data cleaning tasks, and makes working with strings easy. To download a cheat sheet on common stringr functions, click here.

For these examples we will be using a vector of premade strings called strings, and not the diamonds data set.

string_vector <- c("Suny", "New Paltz", "analytics", "club")
# Make all values of string_vector entirely lowercase
str_to_lower(string_vector)
## [1] "suny"      "new paltz" "analytics" "club"
# Make all values of string_vector entirely uppercase
str_to_upper(string_vector)
## [1] "SUNY"      "NEW PALTZ" "ANALYTICS" "CLUB"
# Split the values in string_vector on a space (notice the output)
str_split(string_vector, pattern = " ")
## [[1]]
## [1] "Suny"
## 
## [[2]]
## [1] "New"   "Paltz"
## 
## [[3]]
## [1] "analytics"
## 
## [[4]]
## [1] "club"
# What is the length of each string in string_vector?
str_length(string_vector)
## [1] 4 9 9 4
# Replace analytics in string_vector with "business analytics"
str_replace(string_vector, pattern = "analytics", replacement = "business analytics")
## [1] "Suny"               "New Paltz"          "business analytics"

Stringr also allows us to use regular expressions, which allows you to define and search for a specific pattern. You can find more information on regular expressions in the cheat sheet for stringr above, or by downloading the regular expression cheatsheet. # Detect strings with a "u" in them

Forecats

Forecats is a package in the tidyverse that helps us deal with categorical variables. Categorical variables have a fixed and known set of possible values, this could be something like sex or education level. This is very useful when doing an analysis, because you will most likely not want to work with characters or strings. You can download a cheat sheet of common forecats functions here. We will be working with only the cut column of the diamonds data in these examples.

# Convert the cut column into a factor variable and look at the levels, do not save the output
diamonds %>% 
    mutate(cut = factor(cut)) %>% 
    select(cut) %>% 
    pull() %>% 
    levels()
## [1] "Fair"      "Good"      "Ideal"     "Premium"   "Very Good"

We want to show the order of the cut variable (Fair -> Ideal) when we create it by making it an ordered factor.

# Create a ordered factor of the cut variable and replace the original cut column
diamonds <- diamonds %>% 
    mutate(cut = factor(cut, levels = c("Fair", "Good", "Very Good", "Premium", "Ideal"), ordered = TRUE))
# Look at the levels of the cut variable using the head() function
head(diamonds$cut)
## [1] Ideal     Premium   Good      Premium   Good      Very Good
## Levels: Fair < Good < Very Good < Premium < Ideal
# From cut in the original dataset, use fct_lump to create a factor variable with n = 2, save as lumped_cut and print the first couple of rows using head()
lumped_cut <- diamonds %>% 
    mutate(cut = fct_lump(cut, n = 2)) %>% 
    select(cut) %>% 
    pull()
# Look at levels using head()
head(lumped_cut)
## [1] Ideal   Premium Other   Premium Other   Other  
## Levels: Premium < Ideal < Other

Note that since cut is an ordered factor, it would not be a good idea to lump them together (it has other greater than premium and ideal). You will lose valuable information about the different cuts of diamonds. Use fct_lump when you have a large number of levels.

# Recode the levels in cut_factor so that everything is the same except Ideal becomes best, save as cut_recode and use head() to look at it
cut_recode <- diamonds$cut %>% 
    fct_recode("Best" = "Ideal")

Ggplot2

Ggplot2 allows you to create decorative graphs in R. It is one of the most popular packages for creating data visualizations. You can find a cheat sheet for common ggplot2 graphs and functions here. The cheat sheet also provides what types of graphs you can use when working with different variables, which comes in handy in case you forget.

# Create a scatterplot of carat and price
ggplot(diamonds, aes(x = carat, y = price)) +
    geom_point()

A lot of the points in the graph seem to be overlapping, we can fix this by using geom_jitter instead of geom_point and also decrease the visibility of each point slightly by decreasing the alpha. We can also change the axis names and add a simple title.

# Use geom_jitter to create a scatterplot and add new axis labels and a title, use alpha = .2
ggplot(diamonds, aes(x = carat, y = price)) +
    geom_jitter(alpha = .2) +
    labs(title = "Price vs Carat", x = "Carat", y = "Price")

# Create a bar graph of cut and price
ggplot(diamonds, aes(x = cut, y = price)) +
    geom_col()

This plot is okay, but with ggplot we can create much nicer looking graphs. Lets flip the coordinates, give the graph some color, fix the axis names and give the graph a title.

# Create the graph above but add color, new axis names, and a title, flip axis
ggplot(diamonds, aes(x = cut, y = price)) +
    geom_col(fill = "light blue") +
    coord_flip() +
    labs(title = "Price vs Cut", y = "Price", x = "Cut")

This graph is good, but lets just say we want the lowest price to be on the top of the graph for some reason. We can do this using forecats fct_rev function.

# Make the same plot but with the order of cut from lowest to highest
ggplot(diamonds, aes(x = fct_rev(cut), y = price)) +
    geom_col(fill = "light blue") +
    coord_flip() +
    labs(title = "Price vs Cut", y = "Price", x = "Cut")
